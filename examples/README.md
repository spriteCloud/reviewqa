# Examples — what `reviewqa probe` produces

Each subdirectory is the complete output of running

```bash
reviewqa probe --url <site> --dry-run
```

Generated by the **v0.64 binary**; no hand-editing. Re-emit on every
release via `scripts/refresh-examples.sh`.

## What's in each subdirectory

The same shape every probe run produces — a runnable Playwright + Gherkin
project with stakeholder docs alongside:

```
<site>/
├── playwright.config.ts             # defineBddConfig() + multi-project + cross-browser
├── package.json                     # depends on playwright-bdd + @axe-core/playwright
├── tsconfig.json
├── .github/workflows/e2e.yml
└── tests/e2e/
    ├── features/                    # Gherkin contracts (executable via playwright-bdd)
    │   └── <site>-<journey>.feature
    ├── steps/                       # Given/When/Then → lib/steps.ts
    ├── lib/                         # Steps API helper verbs
    ├── api/                         # API contract spec per <form action> (10 tests each)
    ├── contract/                    # OpenAPI / GraphQL contract specs when schema discovered (9 tests each)
    ├── a11y/                        # axe-core + landmarks + keyboard (a11y trio: 5 tests each, on EVERY page)
    ├── responsive/                  # mobile / tablet / desktop viewports
    ├── perf/                        # page load < SLO
    ├── visual/                      # toHaveScreenshot baselines + visual-states
    ├── security/                    # baseline security headers (per origin)
    ├── health/                      # /health, /healthz, /ready, /status (per origin)
    ├── observability/               # x-request-id, server-timing, traceparent (per origin)
    ├── i18n/                        # locale switching + always-on <html lang> check
    ├── mobile/                      # 4-device matrix × 2 orientations (capped)
    ├── network/                     # slow-3g, offline, 429 (capped)
    ├── storage/                     # localStorage/cookie corruption (capped)
    ├── print/                       # @media print rendering (capped)
    ├── race/                        # double-submit (gated on <form>)
    ├── clipboard/                   # paste sanitization (gated on text input)
    ├── http-chains/                 # 3xx chains, 410, 429+retry-after (per origin)
    ├── file-upload/                 # gated on <input type=file>
    ├── iframe/                      # gated on <iframe>
    ├── date-edges/                  # gated on <input type=date>
    ├── pwa/                         # gated on <link rel=manifest>
    ├── history-depth/               # back→back→forward
    ├── touch/                       # long-press, swipe, pinch (5 gestures)
    ├── dragdrop/                    # gated on [draggable]
    ├── auth-expiry/                 # session expiry mid-flow
    ├── integration/                 # api stub
    ├── integration-db/              # DB scaffold
    ├── integration-cache/           # cache scaffold
    ├── integration-obs/             # observability scaffold
    ├── integration-auth/            # auth scaffold
    ├── graphql/                     # always-attempt /graphql introspection (6 tests)
    ├── webhook/                     # always-attempt webhook stub (6 tests)
    ├── *-fuzz.spec.ts               # plain Playwright TS fuzz/negative specs
    ├── _fixtures.ts                 # shared test/expect with page-error tracking
    └── docs/
        ├── test-catalogue.md        # stakeholder doc — every journey + priority
        ├── summary.html             # branded HTML deck with priority-mix bar
        └── findings.md              # bug ledger (seeded empty; persists across runs)
```

## Re-generating

These outputs are snapshots committed for reference. Refresh after a
release with the one-shot script:

```bash
# 4 deterministic examples (no LLM needed):
./scripts/refresh-examples.sh

# All 6 examples (requires an OpenAI-compatible endpoint for the 2 AI-on
# ones — Ollama, vLLM, OpenAI, spriteCloud's DGX, anything):
QUAIL_LLM=http://your-endpoint:11434 ./scripts/refresh-examples.sh
```

The script builds the binary, runs `reviewqa probe --dry-run` against each
target site, and splits the multi-file stdout into the appropriate
`examples/<site>/` directory.

## Why these six sites?

See [`docs/reference-sites.md`](../docs/reference-sites.md) for the original
four. They cover a deliberate spread of shapes — JS-heavy docs (playwright.dev),
OSS marketing (gohugo.io), e-commerce list/detail (books.toscrape.com),
non-English content (es.wikipedia.org/Madrid) — so the generator is
exercised against more than one site's vocabulary.

The two AI-on examples complete the picture:
**spritecloud.com** demonstrates the LLM composer adding @llm-composed
journey variants on top of the deterministic baseline; **petstore.swagger.io**
demonstrates the Contract layer fully populated from an OpenAPI 3 doc
(impossible on a marketing site without a schema surface).
