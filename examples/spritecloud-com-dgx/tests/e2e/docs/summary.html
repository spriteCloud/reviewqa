Recommended next steps

How to put this suite to work.

1

Install dependencies

Run npm install in this directory to pull @playwright/test and playwright-bdd, then npx playwright install to fetch the browser binaries.

2

Execute the suite

npx bddgen generates the executable specs from the .feature files. Then npx playwright test runs the full suite. Tag-filter with --grep @priority:critical on every PR; nightly the full set.

3

Triage failures

Open tests/e2e/docs/findings.md after each run. reviewqa ledger update --report playwright-report.json deduplicates failures by (spec, test) and surfaces them risk-weighted by priority — survives across runs.

4

Re-probe on change

Re-run reviewqa probe after feature merges to track coverage drift — new pages and new forms get picked up automatically; removed surfaces drop out.

5

Tailor in the browser

reviewqa serve --workdir . opens a local UI where stakeholders can delete journeys, refine Scenarios via chat, suggest locators from the live DOM, and run individual Scenarios — no IDE required.
