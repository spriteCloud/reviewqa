What it does

Deterministic where it matters. Humanized where it helps.

01

Gherkin-first scaffolding

Regex and heuristic extractors map live URLs to user journeys, then emit .feature files in Given/When/Then. Stakeholders read the contract; playwright-bdd runs it.

02

Journeys, priorities, tags

Every test name carries @journey:<kind> and @priority:<level>. Run --grep @priority:critical on every PR; standard nightly; nice-to-have weekly.

03

API contracts + UI flows

Every form on the site gets a sibling *.api.spec.ts — happy, missing-required, malformed-email, oversized, wrong-method — decoupled from the UI happy-flow.

04

Stakeholder docs included

Each run drops tests/e2e/docs/test-catalogue.md (every page, every journey, every priority) and a branded summary.html with a priority-mix bar. Reviewers don't need to read code.

05

Bug-discovery ledger

Failed tests don't die in CI logs — quail ledger update merges them into findings.md, deduped by (spec, test), risk-weighted by priority, surviving across runs.

06

Self‑healing locators

Playwright locator failures get rewritten against the live DOM. Failed CI runs become passing CI runs without manual edits.

07

Quality battery on every probe

Every probe emits accessibility (axe-core), responsive viewports, perf SLO, security headers, health probes, observability headers, visual baselines (toHaveScreenshot), and — when the site advertises them — OpenAPI / GraphQL contract + webhook + i18n specs.

08

gRPC contracts from .proto diffs

Drop a .proto file in your PR and quail emits per-RPC contract tests classified by streaming shape (unary / server-stream / client-stream / bidi) using @grpc/grpc-js.

09

Cross‑platform — Linux, macOS, Windows

Single Go binary, no CGO. AGPL-3.0. v1.0.0 ships quail-review_linux_amd64.tar.gz, quail-review_darwin_arm64.tar.gz, quail-review_windows_amd64.zip, and arm64 variants. Same engine in CI and on your laptop.

Coverage knobs

Dial breadth vs. depth without forking the config.

# quick CI smoke (8 pages, 1 journey per kind, 3 fuzz specs)
quail probe --url https://shop.example.com --coverage breadth

# default — current behaviour (20 pages, 3 journeys per kind, 5 fuzz)
quail probe --url https://shop.example.com

# high-stakes audit (50 pages, 6 journeys per kind, 10 fuzz)
quail probe --url https://shop.example.com --coverage depth

# focused, prompt-driven, runs and bundles an evidence ZIP
quail prompt "verify the checkout flow" \
  --url https://shop.example.com --evidence
Install

One binary. Linux, macOS, Windows.

Drop the action into a workflow, or grab the binary for local probe runs. AGPL-3.0; full source at spriteCloud/quail-review.

# macOS / Linux — install once (amd64 or arm64)
curl -fsSL "https://github.com/spriteCloud/quail-review/releases/latest/download/quail-review_$(uname -s | tr '[:upper:]' '[:lower:]')_$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/').tar.gz" | sudo tar -xz -C /usr/local/bin quail-review

# Windows (PowerShell) — install once
iwr -OutFile $env:TEMP\qr.zip https://github.com/spriteCloud/quail-review/releases/latest/download/quail-review_windows_amd64.zip
Expand-Archive -Force $env:TEMP\qr.zip "$env:USERPROFILE\quail-review"

# probe a live URL — emits a full Playwright + Gherkin suite
quail-review probe --local --url https://your-site.example.com
Drop‑in workflow

One step. Pure Go. No CGO.

Bring any OpenAI‑compatible endpoint — ollama, vLLM, a DGX‑hosted model, or the public API. Leave the key empty and you still get deterministic scaffolds.

# .github/workflows/quail.yml
name: quail
on: pull_request
jobs:
  quail:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: spriteCloud/quail-review@v1
        with:
          openai-api-key: ${{ secrets.OPENAI_API_KEY }}
          heal-mode: on-failure
Common questions

FAQ

Does quail send my source code to a third-party LLM?

Only when you configure an inference endpoint and your `OPENAI_API_KEY` is set. Even then, the LLM is restricted to rewriting test titles and step comments inside a deterministic scaffold. The PR diff is never sent unless you explicitly opt in with `QUAIL_ALLOW_DIFF_TO_LLM=1`.

What happens when the LLM is unavailable or returns garbage?

quail keeps the deterministic scaffold. The LLM rewrite is checked against the original for structural equivalence — same imports, same number of test declarations — and any mismatch is silently discarded.

Can I use my own model instead of OpenAI?

Yes. Any OpenAI-compatible endpoint works — ollama, vLLM, a DGX-hosted model, or a private gateway. Point `OPENAI_BASE_URL` at it and set `QUAIL_MODEL` to the model name.
